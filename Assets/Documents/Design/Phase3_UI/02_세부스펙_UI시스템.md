# 세부 스펙: UI 시스템

## 1. 목적
- 일관된 UI 아키텍처
- 반응형 디자인
- 상태 기반 UI 업데이트

## 2. UI 계층 구조

```
Canvas/
├── HUD/ (항상 표시)
│   ├── ScoreText
│   ├── TouchCountText
│   └── CharacterDisplay
├── TopBar/ (좌측 상단 5 개)
│   ├── SettingButton
│   ├── CharacterColorButton
│   ├── BackgroundButton
│   ├── SpeedBoostButton
│   └── ItemButton
├── Windows/ (팝업)
│   ├── SettingWindow
│   ├── ShopWindow
│   └── AchievementWindow
└── Dialogs/ (알림)
    ├── ConfirmDialog
    └── NotificationDialog
```

## 4. 상단 UI 5 개 상세

### 4.1 SettingButton
- 클릭: SettingWindow 토글
- 데이터: 사운드 설정, 볼륨 값

### 4.2 CharacterColorButton
- 클릭: 색상 선택 팝업
- 데이터: 현재 색상, 언락된 색상 목록

### 4.3 BackgroundButton
- 클릭: 배경 선택 팝업
- 데이터: 현재 배경, 언락된 배경 목록

### 4.4 SpeedBoostButton
- 클릭: 2 배속 활성화
- 데이터: 남은 시간, 쿨다운 상태
- 표시: 비활성화 시 회색

### 4.5 ItemButton
- 클릭: 아이템 사용
- 데이터: 보유 개수
- 표시: 개수 배지

## 5. SettingWindow

### 5.1 사운드 설정
- BGM 토글 (on/off)
- BGM 슬라이더 (0-100)
- SFX 토글 (on/off)
- SFX 슬라이더 (0-100)
- 저장 버튼

### 5.2 데이터 저장
- PlayerPrefs 사용
- JSON 직렬화
- 자동 저장 옵션

## 6. 반응형 디자인
- Canvas Scaler: Scale With Screen Size
- Reference Resolution: 1920x1080
- Anchor 기반 레이아웃

## 7. 애니메이션
- Window: Fade In/Out (0.3s)
- Button: Scale On Click (0.1s)
- Notification: Slide In (0.2s)
